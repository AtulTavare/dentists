Add an Apple-style scroll-scrubbed frame-sequence hero animation to this project using GSAP ScrollTrigger, Lenis smooth scroll, and a DPR-aware canvas. Place all frame images in the `public/frames/` folder named `frame_001.webp`, `frame_002.webp`, ... up to `frame_NNN.webp` (3-digit zero-padded). The canvas renders frames with COVER sizing (fill entire viewport, crop overflow) using `Math.max(cw/imgW, ch/imgH)` centered, and the canvas element is sized to `offsetWidth × offsetHeight` with `ctx.scale(dpr, dpr)` for high-DPI displays. A 300vh pinned scroll area is created with GSAP ScrollTrigger (`pin: true`, `scrub: 1`, `start: "top top"`, `end: "+=300vh"`, `onUpdate` passes `progress` to the canvas component). Inside the pin area, place 3 content sections each as `absolute inset-0` within a viewport-height overlay, toggled by progress thresholds: section 1 visible at `progress < 1/3`, section 2 at `1/3 <= progress < 2/3`, section 3 at `progress >= 2/3`. Section 1 contains a hero heading and CTA left-aligned, section 2 a single heading at center-bottom, section 3 content split far-left and far-right only (nothing in center). The canvas wrapper uses `absolute -top-[NAVBAR_HEIGHT] left-0 w-full h-[calc(100vh+NAVBAR_HEIGHT)]` to extend behind the fixed navbar (NOT `position: fixed` — Lenis transforms break fixed positioning). Three `h-screen` empty spacer divs below the content overlay provide the 300vh natural height for the pin. On mobile (<768px) show frame 0 as a static image with no scroll interaction; on `prefers-reduced-motion: reduce` show the middle frame only. Connect Lenis to GSAP via `lenis.on("scroll", () => ScrollTrigger.update())` and `gsap.ticker.add((t) => lenis.raf(t * 1000))` with `gsap.ticker.lagSmoothing(0)`. Add `data-lenis-prevent` to any `position: fixed` elements. Create a reusable `<Reveal>` component that wraps children with an IntersectionObserver (threshold 0.15), fading from `opacity:0 translate(Xpx,Ypx)` to `opacity:1 translate(0,0)` over 0.7s with a configurable delay. Apply `<Reveal>` to all non-pinned page sections (below the 300vh pin area). Use these exact components:

**ScrollCanvas.tsx** — Accepts `progress` prop. Preloads `TOTAL_FRAMES` images, detects mobile and reduced motion, draws the correct frame on each `progress` update. Canvas has `ctx.scale(dpr, dpr)` applied after sizing. Frame source path format: `/frames/frame_${String(i+1).padStart(3,"0")}.webp`.

**PinnedSections.tsx** — Creates the `ScrollTrigger.create` with pin. Contains the canvas wrapper (absolute offset by navbar height), the 3-section content overlay, and 3 empty `h-screen` spacers. Content visibility driven by `progress` thresholds. Accepts custom JSX content for each section.

**LenisProvider.tsx** — Client component that initializes Lenis on mount and connects it to GSAP ticker + ScrollTrigger. Wraps page body.

**Reveal.tsx** — Client component wrapping children with IntersectionObserver fade+slide-in animation.

Place `<LenisProvider>` in `layout.tsx` wrapping the entire body. Place `<PinnedSections>` at the top of your homepage, followed by any other page sections wrapped in `<Reveal>`. The navbar must be a fixed element with `data-lenis-prevent` attribute and a spacer below it (`h-20` or matching its height). The canvas accounts for this spacer via its negative top offset. That's it — put frames in the folder, run `npm install gsap lenis`, paste the component code, and the scroll animation works.
